Problem

Every couple hours huge amount of data (single package being up to ~4 GB) has been exported by the client to multiple external vendors. As the export network rose efficency problems arrised and couldn't keep up with marketing requirements.

Solution

Since transformation were working and were too complex to re-create from the scratch, new application design had to reuse existing components. Design phase took that into mind and provided highly modular, stateless solution which allowed not only for easy clustering, but also allowed for use cache component for qualified calculations.

Design Trivia was created with MQ as a base point.
